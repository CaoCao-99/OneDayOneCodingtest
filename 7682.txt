틱택토 다국어

문제
{
틱택토 게임은 두 명의 사람이 번갈아가며 말을 놓는 게임이다. 게임판은 3×3 격자판이며, 처음에는 비어 있다. 

두 사람은 각각 X 또는 O 말을 번갈아가며 놓는데, 반드시 첫 번째 사람이 X를 놓고 두 번째 사람이 O를 놓는다. 

어느 때든지 한 사람의 말이 가로, 세로, 대각선 방향으로 3칸을 잇는 데 성공하면 게임은 즉시 끝난다. 게임판이 가득 차도 게임은 끝난다.

게임판의 상태가 주어지면, 그 상태가 틱택토 게임에서 발생할 수 있는 최종 상태인지를 판별하시오.
}

입력
{
입력은 여러 개의 테스트 케이스로 이루어져 있다. 각 줄은 9개의 문자를 포함하며, 'X', 'O', '.' 중 하나이다.

'.'은 빈칸을 의미하며, 9개의 문자는 게임판에서 제일 윗 줄 왼쪽부터의 순서이다. 입력의 마지막에는 문자열 "end"가 주어진다.
}

출력
{
각 테스트 케이스마다 한 줄에 정답을 출력한다. 가능할 경우 "valid", 불가능할 경우 "invalid"를 출력한다.
}


예제 입력 1 
{
XXXOO.XXX
XOXOXOXOX
OXOXOXOXO
XXOOOXXOX
XO.OX...X
.XXX.XOOO
X.OO..X..
OOXXXOOXO
end
}


예제 출력 1 
{
invalid
valid
invalid
valid
valid
invalid
invalid
invalid
}

풀이 방법
{
    단순한 구현 문제이다.

    이런 구현문제는 조건문을 잘 설정해야 올바른 정답을 낼 수 있다.

    본 문제는 주어진 예시가 최종 종료 단계인지 알아내는 문제인데 이 말은 주어진 예시가 정상적인 상황이더라도 끝나는 상황이 아니라면 invalid를 출력해야 한다.

    따라서, 가장 먼저 말이 안되는 경우를 걸러내고 말이 되는 상황에서 종료가 가능한지 따지면 문제를 해결할 수 있다.

    말이 안되는 경우는 다음과 같다.

    1. X표시가 O보다 적은 경우 or X표시가 O보다 2개이상 많은 경우는

    2. 두 종류의 문양이 각각 줄을 만든 경우

    위의 경우를 제외하고 가능한 경우는 2가지로 나눌 수 있다.

    1. X표시의 개수와 O표시의 개수가 같은 경우

        1. .이 있는 경우(O표시 한줄 + X표시 없어야 함) -> valid
        
        2. .이 없는 경우 -> 불가능(invalid)



    2. X표시의 개수 = 1 + O표시의 개수

        1. .이 없는 경우(X표시 2줄 가능, X표시 한줄, X표시 0줄, O표시 없어야 함)

        2. .이 있지만 문양이 한줄을 만든 경우(X표시 한줄, O표시 없어야 함)



    위의 조건문을 돌리면 문제를 해결할 수 있다.
}

느낀 점
{
    이러한 구현 문제는 늘 예외 처리를 어디까지 잘 생각하냐에 달린 것 같다.

    최대한 많은 테스트 케이스를 돌려보며 빼먹은 건 없는 지 생각해 보는 것이 가장 중요한 유형이다.
}
